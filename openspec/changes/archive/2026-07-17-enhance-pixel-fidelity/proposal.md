# Proposal: enhance-pixel-fidelity

## Why

当前图片导入采用最近邻降采样：每个目标像素只取源图区域内的 1 个点，丢弃 99.99% 的信息。对于高分辨率照片，缩小到 32×32 后颜色偏、细节糊，像素还原度差。

用户希望像素化后的图片更贴近原图样式，特别是细节上的像素还原。核心问题是**抗混叠缺失**——缩小图像时必须对源区域做聚合统计，单点采样无法代表整个区域。

## What Changes

### 算法升级：双阶段处理

1. **锐化卷积预处理**（降采样前）
   - 对裁切后的源图做拉普拉斯锐化卷积（3×3 核，中心 5 / 四邻 -1）
   - 增强边缘对比度，避免区域平均后边缘被进一步模糊
   - 保护降采样时即将丢失的高频细节

2. **区域平均降采样**（取代最近邻）
   - 每个目标像素 = 源图对应矩形区域内所有像素的 RGBA 均值
   - 等价于 box filter 抗混叠，是缩小图像的理论最优
   - 输出仍为 32×32 硬边纯色块，像素画审美不变

### 性能策略

对大图（cropSize > 1024）先用浏览器双线性缩放到安全尺寸（≤1024），再做锐化+区域平均。控制卷积和区域平均的计算量，避免高分辨率照片卡顿。

### 不改的部分

- 居中方裁逻辑不变
- 输出格式 `PixelGrid = (string|null)[][]` 不变
- 透明像素判定逻辑不变（alpha 加权平均后按阈值判定）
- 零外部依赖不变
- UI / store / 其他组件不变

## Impact

- **修改文件**：`src/utils/imageImport.ts` — 重写 `sampleImage`，新增锐化卷积 + 区域平均降采样
- **不改**：store、Toolbar、App、Icon、CSS、types、constants、其他 utils
- **不改**：OpenSpec image-import spec 的其他 requirements（仅 MODIFIED"图片像素化转换"）

## Non-goals

- 不引入第三方 npm 依赖（RgbQuant / image-js 等暂不引入）
- 不做调色板量化/颜色限制（保留原色，第一版不强制映射有限色集）
- 不做 UI 上的锐化强度调节滑块（内置固定参数，后续可加）
- 不改变导出/编辑/持久化等已有行为
- 不做多算法可选（区域平均 + 锐化为唯一新策略，不保留旧最近邻作为可选）
